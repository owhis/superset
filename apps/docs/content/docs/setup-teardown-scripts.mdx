## Overview

Run commands automatically when creating or deleting workspaces.

Create `.superset/config.json` in your project:

```json
{
  "setup": ["bun install", "cp \"$SUPERSET_ROOT_PATH/.env\" .env"],
  "teardown": ["docker-compose down"],
  "filesToKeep": [".env", ".vscode/settings.json"]
}
```

## How It Works

1. Create workspace â†’ setup commands run
2. Delete workspace â†’ teardown commands run

Commands run sequentially in the workspace directory.

`filesToKeep` runs during workspace bootstrap and copies matching relative paths from the main repository into the new worktree.

Copy behavior:
- Only explicit relative paths are supported
- Existing files in the worktree are preserved (no overwrite)
- Missing paths are skipped
- Absolute and traversal paths are rejected

## Environment Variables

| Variable | Description |
|----------|-------------|
| `SUPERSET_ROOT_PATH` | Path to root repository |
| `SUPERSET_WORKSPACE_NAME` | Current workspace name |

## Examples

**Node.js:**
```json
{ "setup": ["bun install", "cp \"$SUPERSET_ROOT_PATH/.env\" .env"] }
```

**Docker:**
```json
{
  "setup": ["docker-compose up -d", "bun run db:migrate"],
  "teardown": ["docker-compose down -v"]
}
```

## User Overrides

Override project setup/teardown scripts without modifying the repo by placing a `config.json` in your home directory:

```
~/.superset/projects/<project-id>/config.json
```

Where `<project-id>` is your project's unique ID in Superset (visible in the app's project settings).

### Priority Order

Config is resolved with this priority order for each key (`setup`, `teardown`, `run`, `filesToKeep`):
1. `~/.superset/projects/<project-id>/config.json` â€” user override
2. `<worktree>/.superset/config.json` â€” worktree-specific
3. `<repo>/.superset/config.json` â€” project default

Higher-priority configs override only the keys they define. Missing keys inherit from lower-priority configs.

### Examples

**Custom setup script:**
```json
{
  "setup": ["~/.superset/projects/<project-id>/setup.sh"],
  "teardown": ["~/.superset/projects/<project-id>/teardown.sh"]
}
```

**Skip setup entirely:**
```json
{ "setup": [], "teardown": [] }
```

## Local Config

Extend or override your team's committed scripts without modifying the repo. Create `.superset/config.local.json` alongside `config.json` â€” it's gitignored automatically.

The same merge behavior applies to `filesToKeep`.

### Prepend and/or append

Add steps before or after the team's scripts:

```json
{
  "setup": {
    "before": ["echo 'running pre-setup'"],
    "after": ["./.superset/my-post-setup.sh"]
  },
  "filesToKeep": {
    "after": [".env.local"]
  },
  "teardown": {
    "after": ["./.superset/my-cleanup.sh"]
  }
}
```

Result: `before` â†’ committed setup scripts â†’ `after`. Keys you don't specify pass through unchanged.

### Override entirely

Use a plain array to replace the team's scripts completely:

```json
{
  "setup": ["./.superset/my-custom-setup.sh"]
}
```

### Mix and match

Override one key while extending the other:

```json
{
  "setup": { "after": ["bun run my-extra-step"] },
  "teardown": ["my-custom-teardown.sh"]
}
```

### Resolution order

The local config merges on top of whichever base config wins the [priority order](#priority-order). If a `config.local.json` exists in both the worktree and main repo, the worktree's local config takes priority.

## Force Delete

If teardown scripts fail during workspace deletion, you can force-delete to skip the failed teardown and continue cleanup.

When teardown fails:
1. Error toast appears with "Delete Anyway" and "View Logs" buttons
2. Click "Delete Anyway" to force-delete immediately
3. Or click "View Logs" to review the failure, then force-delete from the dialog

## Tips

- Keep setup fastâ€”runs every workspace creation
- Commit `.superset/` to share with team
- Use shell scripts for complex logic: `"setup": ["./.superset/setup.sh"]`
- Use `config.local.json` to add personal steps without touching the team's config
- Use user overrides (`~/.superset/projects/`) to replace scripts entirely per-project
